  1. 获取镜像

    docker pull nginx

  2. 列出镜像

    docker images

  3. 使用镜像启动容器，并返回容器ID

​

  1. 启动后会进入容器内，使用 exit 退出

  2. 提交副本

    docker commit -m "commit" -a "liwb" 0b2616b0e5a8 mynginx:v2

    docker commit -m "提交信息说明" -a "用户信息" 容器ID 镜像的仓库名称：版本号

  3. 可重复第二步查看是否正常

利用Dockerfile 来创建镜像
  1. 编辑Dockerfile

  • 使用 # 来注释

  • FROM 指令告诉Docker使用哪个镜像作为基础

  • 维护者的信息

  • RUN 开头的指令会在创建中运行，比如安装一些软件等

    注意一个镜像不能超过127层

  • 此外还可以利用ADD命令复制本地文件到镜像

  • 用 EXPOSE命令向外部开放端口

  • 用 CMD命令来描述容器启动后运行的程序等

  1. 生成镜像

    docker build -t="myimage:v2" .

  2. 利用新创建的镜像来启动一个容器

    docker run -t -i myimage:v2 /bin/bash

  3. 修改镜像的标签

    docker tag 0b2616b0e5a8 myimage:v3

    docker tag 容器ID 名称：版本

从本地文件系统导入

要从本地文件系统导入一个镜像，可以收用openvz的模板来创建： openvz的模板下载地址为templates。

比如，先下载了一个 ubuntu-14.04 的镜像，之后使用以下命令导入：

cat ubuntu-14.04-x86_64-minimal.tar.gz |docker import - ubuntu:14.04

存出和载入Docker镜像

  1. 存出镜像

    docker save -o nginx.tar nginx

  2. 载入镜像

    docker load --imput nginx.tar

    或者

    docker load < nginx.tar
